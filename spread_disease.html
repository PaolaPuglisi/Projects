Part 1: Barabasi - Albert Graph¶

As frist I generate the Barabasi Graph, these are Scalee - Free graph, hence is a graph whose degree distribution asymptotically follows a power law. The BarabsiAlbert model assumes that the probability that a node attaches to another node is proportional to the degree of the node, in fact this kind of graph is composed by hubs (nodes having high degree that exceed the average degree) followed by smaller degree nodes, each followed by others smaller degree nodes and so on. This property is called preferential attachment, so that a vertex that already has a large number of edges connecting it to other vertices will tend to attract still more edges. That's the reason why this graph is used in social networks, where heavily linked nodes represent well-known people with lots of relations and so when a newcomer enters the community, he is more likely to become acquainted with one of those more visible people rather than with a relative unknown.

Here I built a Barabasi - Albert graph, that start with 1000 nodes having 4 edges and then step by step it adds edges to the nodes having more edges, so that with a probability proportional number of edges. As first I check that it's connected. And then we can see the histogram representing the degree distribution. For the preferential attachment we can see that the most part of nodes few edges and only few has more then 10 nodes.

Exercise 1.2¶

Now we can built a function that simulate a SIR epidemic process on the graph. It takes in input the already created graph, parameter beta and parametes mu. Beta is the probability of taking the desease, mu is the probability of be recovered after the disease, hence that nodes cannot take the disease anymore. As first all nodes are considered as susceptible, then it chooses at random a node and it infect this, in fact it's added to the list of infected people (I). Now with a random probability (taken by an uniform) less then beta it could infect his neighbors still susceptible that become infected. The same happens for the recovered list, in fact the function throws a random a probability and if this is less than mu this nodes could be recovered. At the end the function returns a dictionary having as key the Time, Susceptible nodes, Infected nodes, Recovered nodes, and 'StoI' and 'ItoR' for the two lists of nodes passed from susceptible to infected and from infected to recovered.

class SIR:
    def __init__(self, g, beta, mu):
        self.g = g
        self.beta = beta
        self.mu = mu
        
    def run(self, seed=[], num_steps = 1, sentinels = []):
        if len(seed) == 0:
           
            seed = [np.random.choice(self.g.nodes())]
        
        I_set = set(seed)
        S_set = set(self.g.nodes()).difference(I_set)
        R_set = set()
        
        t = 0
        
        StoI = set(seed)
        ItoR = set()
        
        while len(I_set) > 0:
            for i in I_set.copy():
                for s in set(self.g.neighbors(i)).intersection(S_set).copy():
                    if np.random.uniform() < self.beta:
                        S_set.remove(s)
                        I_set.add(s)
                        StoI.add(s)
                        
                      
                if np.random.uniform() < self.mu:
                    I_set.remove(i)
                    R_set.add(i)
                    ItoR.add(i)
    
            t += 1
            if t % num_steps == 0 or len(I_set) == 0:
                yield({'t': t, 'S':S_set, 'I':I_set, 'R':R_set, 'StoI':StoI, 'ItoR':ItoR})

Because of the wide variability in the degree of the nodes, there's need to find immunization strategies. In circumstances then where vaccines are scarce, efficient immunization strategies become necessary to preventing more infectious. In this point I modified the Sir function to apply the first strategy immunization: randomly chosen "immunized" nodes. In fact the only difference is that it chooses some random nodes to "vaccinate" so that these cannot be infected.

Ex. 3.1¶

A common approach for targeted immunization studies in scale-free networks is to target the highest degree nodes for immunization. These nodes are the most highly connected in the network, making them more likely to spread the contagion if infected. Immunizing this segment of the network can drastically reduce the impact of the disease on the network.

Here I modified the SIR function in two ways: the first immunizes the first r% of nodes having higher degree (ordered), the second immunizes the first r% of nodes having higher betweenes centrality. These procedure are almost the same because the nodes with higher degree are more central. But betweenness centrality quantifies the number of times a node acts as a bridge along the shortest path between two other nodes. In fact this measure was introduced as a measure for quantifying the control of a human on the communication between other humans in a social network. Vertices that have a high probability to occur on a randomly chosen shortest path between two randomly chosen vertices have a high betweenness.

class SIRsent1:
    def __init__(self, g, r, sentinels, beta, mu):
        self.g = g
        self.beta = beta
        self.mu = mu
        self.r = r
        self.sent= sentinels
    def run(self, seed=[],  num_steps = 1, sane = []):
        
        perc=self.r

Ex 3.3¶

Another strategy is the acquaintance immunization, it tries to target all of the most highly connected nodes for immunization by going through themselves and their neighbors without knowing the full network topology. We don't know anything about the graph, a part some nodes and their friends. So I decided to build a subgraph, having the nodes that I know and their friends. The problem with this point is: how do I decide the people to vaccinate given the not many information that I have? I decide to build a function that takes in input 10% of nodes that I know and a dictionary of the neighbors of the nodes that I know. Suppose that the maximum budget of vaccines that I have is M, the only thing that I can do is to apply the best strategy that I have to slow down the spread of the epimic. So I took at random M nodes and found their neighbors. After this I computed the betweennes centrality in the dict of the nodes and their neighbors("a"), I ordered this values and I simulate the epidemic thanks the function SIRfriendsBeet that takes in input also the list of my friends ("seedout") and the dict of their neighbors ("seedtot"). Obviously in this case the recover rate will be very small because I have only maximum 10% of vaccines. I decided to use also in this case a loop on the percentage of vaccines and to simulate the epidemic 300 times because of with small numbers of recovered it need more loops to converge.
